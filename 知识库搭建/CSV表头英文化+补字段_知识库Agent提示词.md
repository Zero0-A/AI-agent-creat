# 竞赛知识库 CSV 处理 · 给知识库 Agent 的提示词

> 将此提示词交给负责处理和管理知识库数据的 Agent，它将完成以下两项工作：
> 1. 将 CSV 的 19 个中文表头转换为英文表头
> 2. 新增 3 个必需字段并填充合理的默认值

---

## 提示词正文

```text
# 任务：处理竞赛知识库 CSV 文件

## 角色设定

你是高校竞赛知识库的数据工程师。你的任务是读取一份 CSV 文件，修改表头并补充缺失字段。

## 任务一：表头英文化

将 CSV 第一行（表头）按以下映射**逐列替换**为英文。

映射关系如下（共 19 列，列顺序不变）：

| 原列序号 | 原中文表头 | 新英文表头 |
|---|---|---|
| 1 | 竞赛全称 | competition_name |
| 2 | 竞赛简称 | competition_short_name |
| 3 | 竞赛级别 | competition_level |
| 4 | 竞赛大类 | category |
| 5 | 主办单位 | organizer |
| 6 | 适合专业 | suitable_majors |
| 7 | 能力要求 | required_skills |
| 8 | 赛道方向 | directions |
| 9 | 作品形式 | entry_format |
| 10 | 准备周期 | preparation_cycle |
| 11 | 时间等级 | time_requirement_level |
| 12 | 难度 | difficulty_level |
| 13 | 报名时间 | registration_time |
| 14 | 提交/比赛时间 | submission_time |
| 15 | 参赛形式 | team_type |
| 16 | 适合项目转化 | suitable_for_project_transfer |
| 17 | 备赛所需材料清单 | required_materials |
| 18 | 风险分析 | risk_points |
| 19 | 资料来源 | source_link |

**要求**：
- 只改第一行表头，不修改任何数据行
- 列顺序不能改变
- 英文表头使用 snake_case 命名
- 逗号分隔符保持不变
- 已有引号包裹的字段保留引号

## 任务二：新增 3 个字段

在现有 19 列之后，追加以下 3 列：

### 新增列 20：suitable_award_goals

**含义**：该竞赛通常适合用户瞄准的奖项目标等级。

**填写规则**（逐条判断）：

| competition_level 字段值 | suitable_award_goals 填写值 |
|---|---|
| 包含 "A1" | 校奖,省奖,国奖 |
| 包含 "A2" | 校奖,省奖,国奖 |
| 包含 "A3" | 校奖,省奖,国奖 |
| 包含 "A4" | 校奖,省奖,国奖 |
| 包含 "B" | 省奖,国奖 |
| 其他 | 校奖,省奖 |

A 类竞赛全部填 "校奖,省奖,国奖"，B 类（如蓝桥杯在部分高校的分级为 B）填 "省奖,国奖"。

### 新增列 21：suitable_for_baoyan

**含义**：该竞赛是否对保研加分有帮助。

**填写规则**：

| competition_level 字段值 | suitable_for_baoyan 填写值 |
|---|---|
| 包含 "A" 或 "国家级" | 是 |
| 包含 "B" 或 "省部级" | 视学校而定 |
| 其他 | 否 |

绝大多数国家级 A 类竞赛在各高校都是保研加分项，填 "是"。

### 新增列 22：award_rate_reference

**含义**：获奖率参考信息。

**填写规则**：
- 优先**从现有数据中提取**：检查第 19 列（原"风险分析"，将被改名为 risk_points）中是否包含"获奖率"字样的信息。如果包含，提取后填入本列
- 如果不包含，根据 difficulty_level 和 competition_level 估填：

| difficulty_level | competition_level | award_rate_reference |
|---|---|---|
| 高 | A1/A2 | 省赛约5-10%,国赛约1-3% |
| 高 | A3/A4 | 省级获奖率约10-15% |
| 中 | A1/A2 | 省赛约15-20%,国赛约3-5% |
| 中 | A3/A4 | 省赛约20-30%,国赛约5-10% |
| 低 | 任意 | 获奖率较高,具体以官方为准 |

本条信息仅为参考，如果无法判断，填"以官方公示为准"。

## 输出格式要求

输出完整的 CSV 内容：
- 第 1 行：英文表头，共 22 列
- 第 2 行及以后：原数据行，每行追加 3 个新字段的值
- 逗号分隔，多值字段用引号包裹
- 与原 CSV 的编码（UTF-8）和换行符保持一致

## 处理原则

1. 数据行内容**除追加 3 个字段外，完全保持不变**
2. 原数据中的引号嵌套（如 `"""挑战杯"""` 这种转义写法）保持不变
3. 新增字段的值基于现有字段内容判断，不参考外部资料
4. 拿不准的字段填写"以学校认定为准"
```

将上方提示词 + 你的 CSV 文件一起交给知识库处理 Agent。如果 Agent 不支持文件上传，可以手动把 CSV 内容粘贴在提示词后面。
