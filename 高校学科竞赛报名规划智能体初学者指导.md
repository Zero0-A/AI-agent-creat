## 0. 任务一句话总结

> 开发一个面向大学生的“高校学科竞赛报名规划智能体”，能根据用户的专业、年级、技能基础、时间安排、项目经历和目标奖项，**推荐适合的竞赛和赛道**，并**生成合理的备赛规划**。

交付截止：**7 月 15 日以前**。总分 100 分，其中 **第三方隐藏测试集占 20 分**，因此泛化能力很重要。

## 1. 智能体基本概念与文档任务解读

### 1.1 什么是智能体（Agent）？

对初学者，可以这样理解：

> 智能体 = 大语言模型 + 工作流 + 知识库 + 工具节点 + 记忆 + 对话交互。

它和普通聊天机器人最大的区别在于：

| 维度 | 聊天机器人 | 智能体 |
|---|---|---|
| 目标 | 回答问题 | 完成任务 |
| 流程 | 单轮对话 | 多步骤、多节点 |
| 信息 | 模型自带 | 检索知识库 |
| 输出 | 自由文本 | 结构化、可解释 |
| 工具 | 无 | 可调用代码 / API |
| 行为 | 被动 | 信息不足会主动追问 |

### 1.2 文档任务拆解

PDF 中明确要求智能体至少具备以下 7 个核心能力：

| 编号 | 能力 | 说明 |
|---|---|---|
| F1 | 用户画像识别 | 从自然语言中提取结构化字段（专业、年级、技能、时间、目标等） |
| F2 | 竞赛知识库问答 | 基于自建知识库回答竞赛相关问题，不胡编 |
| F3 | 竞赛推荐 | 输出推荐比赛、赛道、理由、风险、准备周期、优先级 |
| F4 | 备赛计划生成 | 阶段划分、任务、时间、材料、分工、风险、冲奖策略 |
| F5 | 多轮追问 | 信息不全时主动追问，而不是随意推荐 |
| F6 | 工作流设计 | 不能只做提示词聊天，要有清晰的节点流程 |
| F7 | 至少一个代码或工具节点 | 必须有真实工程实现，不能纯靠提示词 |

### 1.3 题目真正考察的能力

文档明确指出本题不是算法刷题，而是考察 8 项综合能力：

1. 需求分析能力
2. 资料检索与知识库搭建能力
3. 智能体工作流设计能力
4. 提示词工程能力
5. 基础工程实现能力
6. 测试与迭代能力
7. PPT 汇报与表达能力
8. 自学能力

### 1.4 交付清单（共 9 项）

- 智能体原型链接或本地演示方式
- README 使用说明
- 工作流截图或架构图
- 知识库整理文档
- 主要提示词文档
- 代码或工具节点说明
- 至少 20 条测试记录
- 一份智能体生成的推荐报告样例
- 3-5 页汇报 PPT

### 1.5 评分标准（总分 100）

| 评分项 | 分值 | 关键点 |
|---|---:|---|
| 需求理解 | 8 | 是“竞赛规划”而非闲聊 |
| 资料收集与知识库 | 12 | 来源清楚、结构化 |
| 工作流设计 | 15 | 节点清晰、异常处理、推荐逻辑 |
| 提示词能力 | 10 | 稳定、清晰、少幻觉 |
| 工程实现 | 12 | 有真实代码/工具节点 |
| 自测与迭代 | 8 | 20 条测试 + 优化记录 |
| 隐藏测试成功率 | 20 | 泛化能力，最重要 |
| PPT 与表达 | 10 | 3-5 页清晰汇报 |
| 沟通协作 | 5 | 按时反馈 |

## 5. 分步骤实施指南（基于 Coze 实操）

### 5.1 环境搭建（Coze 路线）

**步骤：**

1. 访问 `coze.cn` 并注册账号；
2. 进入工作空间，点击 **创建智能体**；
3. 命名为“高校竞赛报名规划助手”；
4. 选择编排模式：**单 Agent 模式**（足够）；
5. 选择模型：**豆包 / DeepSeek**（个人版可用）；
6. Temperature 设置为 **0.1-0.3**（让输出稳定）。

### 5.2 知识库搭建（Coze 表格知识库）

**为什么用表格知识库？**

Coze 支持三类知识库：文档、表格、照片。对于结构化竞赛资料，**表格知识库**最适合，因为：

- 按列字段精准检索（专业、能力要求、准备周期等）
- 支持 NL2SQL 查询，比纯文本检索更准
- 维护简单，直接用 Excel 编辑

**步骤：**

1. 在 Coze 资源中创建“表格知识库”；
2. 准备 Excel 文件，表头如下：

```text
竞赛名称,赛道方向,适合专业,能力要求,作品形式,组队要求,报名时间,提交时间,准备周期,冲奖难度,适合已有项目转化,风险点,资料来源
```

3. 填入 10-15 行真实竞赛数据，参考来源：
   - 中国国际大学生创新大赛
   - 挑战杯
   - 蓝桥杯
   - 中国大学生计算机设计大赛
   - 全国大学生电子设计竞赛
   - 全国大学生数学建模竞赛
   - RoboMaster
   - 互联网+
   - “互联网+”创新创业大赛
   - “外研社·国才杯”等

4. 上传 Excel，配置索引列（建议：竞赛名称、适合专业、能力要求）；
5. 在智能体中绑定该知识库。

### 5.4 工作流设计（Coze 节点）

推荐节点流程：

```
开始
  ↓
大模型节点（画像提取，输出 JSON）
  ↓
代码节点（信息完整性检查）
  ↓
选择器节点
  ├─ 信息不足 → 大模型节点（生成追问） → 结束
  └─ 信息充足 → 知识库检索节点
                 ↓
              代码节点（匹配评分函数）
                 ↓
              大模型节点（生成推荐结果）
                 ↓
              大模型节点（生成备赛计划）
                 ↓
              大模型节点（结果自检）
                 ↓
              结束
```

**每个节点职责：**

| 节点 | 类型 | 职责 |
|---|---|---|
| 画像提取 | 大模型节点 | 把自然语言转 JSON |
| 完整性检查 | 代码节点 | 判断必填字段是否缺失 |
| 追问生成 | 大模型节点 | 针对缺失字段生成 3-6 个问题 |
| 知识库检索 | 知识库检索节点 | 找候选竞赛 |
| 匹配评分 | 代码节点 | 给每个候选打分排序 |
| 推荐生成 | 大模型节点 | 生成推荐说明 |
| 备赛计划 | 大模型节点 | 生成阶段计划 |
| 结果自检 | 大模型节点 | 检查输出完整性 |

### 5.5 代码节点示例（Python）

#### 5.5.1 信息完整性检查

```python
async def main(args):
    profile = args.params.get("profile", {}) or {}
    required = ["major", "grade", "skills", "available_time", "award_goal"]
    missing = [k for k in required if not profile.get(k)]
    return {
        "is_complete": len(missing) == 0,
        "missing_fields": missing
    }
```

#### 5.5.2 竞赛匹配评分

```python
async def main(args):
    user = args.params.get("user", {})
    competitions = args.params.get("competitions", []) or []
    results = []

    for c in competitions:
        score = 0
        reasons = []

        # 专业匹配
        if user.get("major") in (c.get("suitable_majors") or []):
            score += 25
            reasons.append("专业匹配")

        # 技能匹配
        user_skills = set(user.get("skills") or [])
        required_skills = set(c.get("required_skills") or [])
        matched = user_skills & required_skills
        if matched:
            score += min(len(matched) * 8, 25)
            reasons.append("技能匹配：" + ", ".join(matched))

        # 时间匹配
        if (user.get("available_time_level") or 0) >= (c.get("time_requirement_level") or 0):
            score += 15
            reasons.append("时间基本满足")

        # 项目基础匹配
        if user.get("has_project") and c.get("suitable_for_project_transfer"):
            score += 15
            reasons.append("已有项目可转化")

        # 目标奖项匹配
        if user.get("award_goal") in (c.get("suitable_award_goals") or []):
            score += 10
            reasons.append("目标奖项匹配")

        # 偏好方向匹配
        if user.get("preferred_direction") in (c.get("directions") or []):
            score += 10
            reasons.append("方向偏好匹配")

        results.append({
            "name": c.get("name"),
            "score": score,
            "reasons": reasons
        })

    results.sort(key=lambda x: x["score"], reverse=True)
    return {"ranked": results[:5]}
```

## 6. 常见问题与解决方案

### Q1：智能体直接推荐，不追问

**解决**：在系统提示词中加入硬性规则：

```text
如果【专业、年级、技能、可投入时间、目标奖项】中任意 2 项缺失，禁止推荐，必须先提 3-6 个问题。
```

### Q2：报名时间被胡编

**解决**：
- 知识库统一标注“以官方通知为准”；
- 提示词加入“不得编造报名时间”；
- 结果自检节点检查是否包含具体日期。

### Q3：推荐理由太空泛

**解决**：要求推荐理由必须对应具体匹配字段（专业、技能、时间、项目、目标、偏好）。

### Q4：输出格式不稳定

**解决**：
- 用固定 Markdown 模板；
- 添加结果自检节点；
- Temperature 调到 0.1-0.3。

### Q5：知识库检索不到内容

**解决**：
- 每个竞赛独立一行；
- 字段名统一；
- 加入关键词（算法、硬件、创新创业、PPT 等）；
- 不要上传混乱文本。

### Q6：代码节点不会写

**解决**：先写最简单的匹配评分函数（专业 + 技能 + 时间 + 目标四维度），不必追求复杂算法。

### Q7：测试覆盖不全

**解决**：按 12 类场景每类 1-2 条设计，重点覆盖信息缺失、时间不足、能力单一、专业不匹配、不合理要求、知识库无答案、多轮追问。

### Q8：隐藏测试得分低（泛化能力差）

**解决**：
- 知识库字段要全面，不要只适配自己的测试用例；
- 匹配评分用规则而非硬编码；
- 提示词避免出现具体竞赛名称的硬性绑定。
