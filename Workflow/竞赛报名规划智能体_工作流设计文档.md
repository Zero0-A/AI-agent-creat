### 1.2 两条路径

| 路径       | 触发条件                         | 经过节点                                           | 输出   |
| -------- | ---------------------------- | ---------------------------------------------- | ---- |
| **推荐路径** | 信息完整（`is_complete = true`）   | 画像提取 → 完整性检查 → 知识库检索 → 匹配评分 → 推荐生成 → 备赛计划 → 结束 | 推荐报告 |
| **追问路径** | 信息不完整（`is_complete = false`） | 画像提取 → 完整性检查 → 追问生成 → 结束                       | 追问问题 |

### 3.3 完整性检查节点

**节点类型**：代码节点（Code Node）

**功能**：判断用户画像中的字段完整性，采用「必填硬检查 + 选填软提醒」分级策略。

- **必填字段（5 个）**：缺失则 `is_complete=false`，进入追问路径，阻塞流程
- **选填字段（3 个）**：缺失仅标记为 `has_soft_gap=true`，不阻塞主流程；后续在推荐生成节点末尾以友好方式引导用户补充

**字段分级**：

| 级别 | 字段 | 缺失后果 |
|---|---|---|
| 必填 | `major / grade / skills / available_time / award_goal` | 阻塞 → 追问 |
| 选填 | `preferred_direction / project_experience / existing_assets` | 不阻塞 → 推荐末尾软提醒 |

**输入区配置**（方案 B：8 个顶层平级变量，分别引用画像提取节点对应字段）：

| 变量名 | 类型 | 引用源 |
|---|---|---|
| major | String | 画像提取.major |
| grade | String | 画像提取.grade |
| skills | Array\<String\> | 画像提取.skills |
| available_time | String | 画像提取.available_time |
| award_goal | String | 画像提取.award_goal |
| preferred_direction | Array\<String\> | 画像提取.preferred_direction |
| project_experience | String | 画像提取.project_experience |
| existing_assets | Array\<String\> | 画像提取.existing_assets |

**代码**（方案 C：分级检查）：

```python
async def main(args) -> dict:
    """
    完整性检查节点（方案 C：必填硬检查 + 选填软提醒分级）
    
    必填字段：缺失则阻塞，进入追问
    选填字段：缺失不阻塞，仅作软提醒，在推荐生成节点末尾引导补充
    """
    params = args.params or {}
    
    profile = {
        "major": params.get('major', '') or '',
        "grade": params.get('grade', '') or '',
        "skills": params.get('skills', []) or [],
        "available_time": params.get('available_time', '') or '',
        "award_goal": params.get('award_goal', '') or '',
        "preferred_direction": params.get('preferred_direction', []) or [],
        "project_experience": params.get('project_experience', '') or '',
        "existing_assets": params.get('existing_assets', []) or []
    }
    
    # 必填字段（缺失阻塞流程）
    required_fields = ["major", "grade", "skills", "available_time", "award_goal"]
    # 选填字段（缺失仅作软提醒）
    optional_fields = ["preferred_direction", "project_experience", "existing_assets"]
    
    def is_empty(value):
        if value is None:
            return True
        if isinstance(value, str) and value.strip() == "":
            return True
        if isinstance(value, list) and len(value) == 0:
            return True
        return False
    
    missing_fields = [f for f in required_fields if is_empty(profile[f])]
    soft_missing_fields = [f for f in optional_fields if is_empty(profile[f])]
    
    is_complete = (len(missing_fields) == 0)
    has_soft_gap = (len(soft_missing_fields) > 0)
    
    return {
        "is_complete": is_complete,
        "missing_fields": missing_fields,
        "missing_count": len(missing_fields),
        "soft_missing_fields": soft_missing_fields,
        "soft_missing_count": len(soft_missing_fields),
        "has_soft_gap": has_soft_gap,
        "profile": profile
    }
```

**输出区配置**（7 个字段）：

| 变量名 | 类型 | 说明 |
|---|---|---|
| `is_complete` | Boolean | 必填是否齐全（选择器判断依据） |
| `missing_fields` | Array\<String\> | 必填缺失列表（送追问节点） |
| `missing_count` | Integer | 必填缺失数量 |
| `soft_missing_fields` | Array\<String\> | 选填缺失列表（送推荐生成节点做软提醒） |
| `soft_missing_count` | Integer | 选填缺失数量 |
| `has_soft_gap` | Boolean | 是否有选填缺失（推荐节点条件判断） |
| `profile` | Object | 透传完整画像（含 8 字段） |

**输出示例**：

```json
// 场景 A：必填缺失（典型追问场景）
{
  "is_complete": false,
  "missing_fields": ["available_time"],
  "missing_count": 1,
  "soft_missing_fields": ["preferred_direction", "project_experience", "existing_assets"],
  "soft_missing_count": 3,
  "has_soft_gap": true,
  "profile": {
    "major": "计算机科学与技术",
    "grade": "大二",
    "skills": ["Python", "C++"],
    "available_time": "",
    "award_goal": "省赛三等奖",
    "preferred_direction": [],
    "project_experience": "",
    "existing_assets": []
  }
}

// 场景 B：必填齐全但选填缺失（进入推荐 + 末尾软提醒）
{
  "is_complete": true,
  "missing_fields": [],
  "missing_count": 0,
  "soft_missing_fields": ["project_experience", "existing_assets"],
  "soft_missing_count": 2,
  "has_soft_gap": true,
  "profile": { ... }
}

// 场景 C：全字段齐全（直接推荐，无软提醒）
{
  "is_complete": true,
  "missing_fields": [],
  "missing_count": 0,
  "soft_missing_fields": [],
  "soft_missing_count": 0,
  "has_soft_gap": false,
  "profile": { ... }
}
```

**说明**：

- `is_complete` 是选择器节点的判断依据，**仅由必填字段决定**
- `missing_fields` 告诉追问生成节点"哪些必填字段需要追问"
- `soft_missing_fields` 和 `has_soft_gap` 仅供**推荐生成节点**使用，用于在推荐末尾追加友好引导
- 选填字段缺失**不会阻塞主流程**，避免用户为非关键信息进行多轮回答
- **设计理念**：用户先看到推荐成果（承诺一致性效应），再被引导补充选填，转化率高于前置追问

### 3.4 追问决策节点

**节点类型**：选择器节点（Condition/Branch Node）

**功能**：根据 `check_result.is_complete` 的值，决定走推荐路径还是追问路径。

**配置**：

| 参数   | 值                              |
| ---- | ------------------------------ |
| 条件变量 | `{{check_result.is_complete}}` |

**分支逻辑**：

| 分支       | 条件                                      | 下一节点    |
| -------- | --------------------------------------- | ------- |
| **追问分支** | `{{check_result.is_complete}} == false` | 追问生成节点  |
| **推荐分支** | `{{check_result.is_complete}} == true`  | 知识库检索节点 |

**Coze 选择器配置界面操作**：

1. 拖入"选择器节点"
2. 设置判断条件：
   - Branch 1：`{{nodes.check_result.output.is_complete}}` 等于 `true` → 连接到"知识库检索节点"
   - Branch 2：`{{nodes.check_result.output.is_complete}}` 等于 `false` → 连接到"追问生成节点"
3. 分支连接后，选择器自动生效

***

### 3.5 追问生成节点

**节点类型**：大模型节点（LLM Node）

**功能**：针对缺失的字段，生成 3-6 个自然流畅的追问问题。

**模型配置**：

| 参数          | 建议值                  |
| ----------- | -------------------- |
| 模型          | 豆包 Pro / DeepSeek V3 |
| Temperature | 0.3                  |
| 最大 Token    | 1500                 |

**系统提示词**：

```text
你是一个高校学科竞赛规划助手的追问模块。

【背景】
用户刚刚发了一条消息，想咨询适合参加的学科竞赛。但用户提供的信息不完整，缺少一些关键字段。
你需要根据已提取到的信息和缺失的字段，提出 3-6 个问题来收集完整信息。

【输入】
已提取到的用户信息：
{{profile}}

缺失的字段：
{{check_result.missing_fields}}

【追问规则】
1. 每条追问要自然、友好，像真人对话一样，不要生硬地提问
2. 不要一次性把问题全部抛出，先提出当前最关键的 2-3 个问题
3. 对用户已经提供的信息，在追问中先确认再提问，例如：
   "你是大二计算机专业的学生，会 Python，这很好。为了给你推荐更合适的比赛，还想了解以下信息："
4. 使用换行分隔不同问题，每个问题用"•"开头
5. 语气亲切、鼓励，不要给用户压力

【缺失字段对应的提问模板】
- major: "你目前学的是什么专业呢？"
- grade: "你现在是大几还是研究生？"
- skills: "你目前掌握了哪些技能或工具？（编程语言、算法、设计软件等）"
- available_time: "你每周大概能投入多少时间准备比赛呢？"
- award_goal: "你的目标是什么？想拿省奖、国奖，还是主要为了保研加分？"
- preferred_direction: "你有偏好的方向吗？比如算法方向、创新创业、硬件开发等"

【输出要求】
只输出追问文本，不要添加任何额外的说明或格式标识。
```

**输入变量映射**：

| 工作流变量                             | 节点内变量名          |
| --------------------------------- | --------------- |
| `{{profile}}`                     | profile         |
| `{{check_result.missing_fields}}` | missing\_fields |

**输出变量**：

```text
// 变量名：reply
// 类型：String
```

### 3.7 匹配评分节点

**节点类型**：代码节点（Code Node）

**功能**：对知识库返回的候选竞赛进行多维匹配打分，按分数排序输出 Top 5。

**代码**：

```python
async def main(args):
    profile = args.params.get("profile", {}) or {}
    candidates = args.params.get("candidates", []) or []
    
    # 用户画像提取
    user = {
        "major": (profile.get("major") or "").strip(),
        "skills": [s.strip() for s in (profile.get("skills") or []) if s.strip()],
        "available_time": (profile.get("available_time") or "").strip(),
        "award_goal": (profile.get("award_goal") or "").strip(),
        "preferred_direction": [d.strip() for d in (profile.get("preferred_direction") or []) if d.strip()],
        "has_project": bool(profile.get("project_experience") or profile.get("existing_assets")),
        "project_experience": (profile.get("project_experience") or "").strip()
    }
    
    results = []
    
    for comp in candidates:
        score = 0
        reasons = []
        details = {}
        
        # ============ 维度1：专业匹配（25分） ============
        major = user["major"]
        suitable_majors = (comp.get("suitable_majors") or "")
        
        if major and suitable_majors:
            # 如果适合专业包含"全专业"，直接给满分
            if "全专业" in suitable_majors or "全专业" in suitable_majors:
                score += 25
                reasons.append("支持全专业参赛")
                details["major"] = 25
            # 如果用户专业在适合专业列表中
            elif any(m in suitable_majors for m in [major, major[:2]]):
                # 精确匹配25分，部分匹配18分
                if major in suitable_majors:
                    score += 25
                    reasons.append(f"专业完全匹配：{major}")
                    details["major"] = 25
                else:
                    score += 15
                    reasons.append(f"专业部分匹配：{major}属于{suitable_majors}覆盖范围")
                    details["major"] = 15
            else:
                details["major"] = 0
        
        # ============ 维度2：技能匹配（25分） ============
        user_skills = user["skills"]
        req_skills_str = (comp.get("required_skills") or "")
        
        if user_skills and req_skills_str:
            req_skills = [s.strip() for s in req_skills_str.split(",") if s.strip()]
            matched = [s for s in user_skills if any(r.contains(s) or s.contains(r) for r in req_skills)]
            
            if matched:
                skill_score = min(len(matched) * 8, 25)
                score += skill_score
                reasons.append(f"技能匹配：{'、'.join(matched)}")
                details["skills"] = skill_score
            else:
                details["skills"] = 0
        else:
            details["skills"] = 0
        
        # ============ 维度3：时间匹配（15分） ============
        time_str = user["available_time"]
        if time_str:
            # 识别时间充足度的大致关键词
            has_long_time = any(k in time_str for k in ["暑假", "寒假", "长期", "每天", "20小时", "30小时", "充足"])
            has_short_time = any(k in time_str for k in ["两周", "一周", "很短", "紧张", "没时间"])
            
            try:
                time_level = int(comp.get("time_requirement_level") or 2)
            except:
                time_level = 2
            
            if has_long_time:
                score += 15
                reasons.append("时间充足，适合各类比赛")
                details["time"] = 15
            elif has_short_time and time_level <= 2:
                score += 10
                reasons.append("时间较紧，建议选择周期适中的比赛")
                details["time"] = 10
            elif has_short_time:
                score += 5
                reasons.append("时间紧张，需注意备赛周期")
                details["time"] = 5
            else:
                score += 10
                details["time"] = 10
        else:
            details["time"] = 0
        
        # ============ 维度4：项目基础匹配（15分） ============
        if user["has_project"]:
            project_transfer = (comp.get("suitable_for_project_transfer") or "")
            if "是" in project_transfer:
                score += 15
                reasons.append("已有项目适合转化参赛")
                details["project"] = 15
            elif "部分" in project_transfer:
                score += 8
                reasons.append("项目可部分转化参赛")
                details["project"] = 8
            else:
                details["project"] = 0
        
        # ============ 维度5：目标奖项匹配（10分） ============
        award = user["award_goal"]
        suitable_awards = (comp.get("suitable_award_goals") or "")
        
        if award and suitable_awards:
            if award in suitable_awards:
                score += 10
                reasons.append(f"目标奖项匹配：{award}")
                details["award"] = 10
            elif "国奖" in award and ("国奖" in suitable_awards or "省奖" in suitable_awards):
                score += 7
                reasons.append("目标奖项基本匹配")
                details["award"] = 7
            else:
                details["award"] = 0
        else:
            details["award"] = 0
        
        # ============ 维度6：方向偏好匹配（10分） ============
        user_dirs = user["preferred_direction"]
        comp_dirs_str = (comp.get("directions") or "")
        
        if user_dirs and comp_dirs_str:
            comp_dirs = [d.strip() for d in comp_dirs_str.split(",") if d.strip()]
            matched_dirs = [d for d in user_dirs if any(c.contains(d) or d.contains(c) for c in comp_dirs)]
            
            if matched_dirs:
                dir_score = min(len(matched_dirs) * 5, 10)
                score += dir_score
                reasons.append(f"方向偏好匹配：{'、'.join(matched_dirs)}")
                details["direction"] = dir_score
            else:
                details["direction"] = 0
        else:
            details["direction"] = 0
        
        # 总分数提示
        if score >= 80:
            level = "强烈推荐"
        elif score >= 60:
            level = "推荐"
        elif score >= 40:
            level = "可考虑"
        else:
            level = "备选"
        
        results.append({
            "competition_name": comp.get("competition_name", ""),
            "total_score": score,
            "max_score": 100,
            "recommendation_level": level,
            "reasons": reasons,
            "details": details,
            # 保留原始竞赛信息用于后续生成推荐理由
            "competition_info": {
                "level": comp.get("competition_level", ""),
                "category": comp.get("category", ""),
                "required_skills": comp.get("required_skills", ""),
                "directions": comp.get("directions", ""),
                "entry_format": comp.get("entry_format", ""),
                "registration_time": comp.get("registration_time", ""),
                "submission_time": comp.get("submission_time", ""),
                "team_type": comp.get("team_type", ""),
                "preparation_cycle": comp.get("preparation_cycle", ""),
                "difficulty_level": comp.get("difficulty_level", ""),
                "risk_points": comp.get("risk_points", ""),
                "required_materials": comp.get("required_materials", ""),
                "suitable_award_goals": comp.get("suitable_award_goals", ""),
                "suitable_for_baoyan": comp.get("suitable_for_baoyan", "")
            }
        })
    
    # 按总分降序排列
    results.sort(key=lambda x: x["total_score"], reverse=True)
    
    # 返回 Top 5
    top5 = results[:5]
    
    return {
        "ranked": top5,
        "total_count": len(results),
        "user_summary": {
            "major": user["major"],
            "grade": profile.get("grade", ""),
            "skills": user["skills"],
            "award_goal": user["award_goal"],
            "has_project": user["has_project"]
        }
    }
```

**说明**：

- 由于 Coze 结束节点只能有一个输出来源，需要用一个**变量聚合节点**或**额外的大模型节点**来处理两条路径的汇合
- 或者将结束节点前的两个路径各接一个结束节点，由 Coze 自动最终输出
- 推荐方案：在结束节点前再接一个**大模型节点**（结果自检节点），将推荐报告和备赛计划拼接后输出

## 4. 数据流与变量映射

### 4.1 完整数据流

```text
开始节点 ──query──▶ 画像提取节点 ──profile──▶ 完整性检查节点 ──check_result──▶ 选择器节点
                                                                                  │
                          ┌─────────────────────────────────────────────────────────┤
                          │                                                         │
                     is_complete=false                                        is_complete=true
                          │                                                         │
                          ▼                                                         ▼
                    追问生成节点                                             知识库检索节点
                          │                                                         │
                     reply ←───┐                                         candidates───▶ 匹配评分节点
                                │                                                         │
                                │                                                   ranked───▶ 推荐生成节点
                                │                                                         │
                                │                                             recommendation───▶ 备赛计划节点
                                │                                                         │
                                │                                                     plan
                                │                                                         │
                                └──── 结束节点 ◄──────────────────────────────────────────┘
```

### 4.2 变量类型汇总

| 变量名              | 类型                | 节点间传递方式         |
| ---------------- | ----------------- | --------------- |
| `query`          | String            | 直接传递            |
| `profile`        | Object (JSON)     | 节点输出 → 节点输入     |
| `check_result`   | Object (JSON)     | 节点输出 → 选择器和追问节点 |
| `reply`          | String            | 追问路径 → 结束节点     |

| `plan`           | String (Markdown) | 备赛计划节点 → 结束节点   |

## 5. 异常处理设计

### 5.1 知识库检索为空

**可能原因**：

- 用户专业非常冷门，知识库中没有匹配
- 检索查询构建不当

**处理方式**：

在匹配评分代码节点中增加空数据处理：

```python
if not candidates or len(candidates) == 0:
    return {
        "ranked": [],
        "total_count": 0,
        "error": "未找到匹配的竞赛",
        "user_summary": { ... }
    }
```

在推荐生成节点的提示词中增加空结果处理：

```text
如果评分结果为空或没有推荐竞赛，请友好地告诉用户：目前知识库中暂未找到与
用户情况完全匹配的竞赛，建议提供更多信息后再次查询。
```

### 5.2 用户输入无法解析

**可能原因**：

- 用户输入过于简短（如"好的""谢谢"）
- 用户输入与竞赛无关

**处理方式**：

在画像提取节点的提示词中增加：

```text
如果用户输入内容与竞赛咨询完全无关（如打招呼、闲聊），输出：
{
  "error": "unrelated_input",
  "message": "请告诉我你的专业、年级、技能等信息，我来为你推荐适合的学科竞赛。"
}
```

在完整性检查节点中检测 `error` 字段，直接走追问路径。

### 5.3 代码节点异常

**可能原因**：

- JSON 解析失败
- 字段类型不匹配

**处理方式**：

代码节点增加 try-except：

```python
try:
    profile = args.params.get("profile", {}) or {}
    # ... 正常逻辑
except Exception as e:
    return {
        "error": True,
        "error_message": str(e),
        "is_complete": False,
        "missing_fields": []
    }
```

## 工作流 Coze 操作速查

### 创建步骤

1. 进入 Coze 工作空间 → 工作流 → 创建工作流
2. 命名：`高校竞赛报名规划工作流`
3. 按本文档 [第 3 节](#3-节点详解) 依次拖入并配置 10 个节点
4. 配置好节点间的连线（开始→画像提取→完整性检查→选择器→...）
5. 保存并发布工作流

### 在智能体中引用

1. 进入智能体编辑页面
2. 找到"技能"区域 → 添加"工作流"
3. 选择刚才创建的工作流
4. 将工作流的输出方式设为"直接回复"
5. 保存并测试
